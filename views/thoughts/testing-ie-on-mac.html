Debugging Internet Explorer on a Mac

If you're a web developer with industry experience, the words "Internet" and "Explorer" spoken together probably make you cringe. Sometimes I have dreams about a world where programmers only write code for other programmers and in this world we only have to be accountable for Mozilla Firefox and Google Chrome. Sadly, this world only exists in my imagination and as developers we need to make sure that the user has a positive experience with the software - even if they're using IE 8.

Aside from the disregard for HTML, CSS and JavaScript standards (this has gotten better with IE versions 10 and 11), another reason web developers dislike IE is that it only runs natively on Microsoft operating systems. Most of the interent runs on Unix based operating systems, so most web programmers prefer to do their work in Unix based environments. Thankfully, Microsoft has recognized this discrepancy and supports virtual machine disk images for testing/debugging IE on non-Microsoft operating systems. Even better is that Github user xdissent has written a Bash script for getting all of the development environments set-up:

  • Make sure you have Oracle VirtualBox installed. If not you can download it for you respective operating system here.
  • Open up a terminal and paste the following command. This will download and set up all of the Microsoft virtual machine disk images into your VirtualBox. Be patient, the process might take a few hours. 
      $ curl -s https://raw.githubusercontent.com/xdissent/ievms/master/ievms.sh | bash
  • After the script is finished executing, you should have have bootable virtual machines for IE versions 6, 7, 8, 9, 10 and 11 installed in your VirtualBox.

With everything installed, you'll still need to make some manual changes to each individual virtual machine that you want to debug on. Specifically, the virtual machine need to be enabled to listen to an outer IP (i.e. localhost on your native operating system):

  • Make sure your network settings for the virtual machine are kept as NAT.
  • Start the virtual machine and open the hosts file in a text editor: 
      C:\windows\system32\drivers\etc\hosts
  • Add the following entry to the hosts file: 
      10.0.2.2    outer
  • Access the website you have running locally on your native operating system via http://10.0.2.2:PORT. For example, if you have a website running locally on port 3000 of your native operating system, you can access it through the virtual machine via http://10.0.2.2:3000

Happy degugging Internet Explorer!
